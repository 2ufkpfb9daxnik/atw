# ドキュメント案内

このフォルダには、タイピング本体と記録画面の実装ドキュメントがあります。

## まず読む順番
1. `docs/main.gd.md`
2. `docs/records.gd.md`

この順番で読むと、保存側（`main.gd`）から表示側（`records.gd`）へ自然につながります。

## ファイル別の目的
- `docs/main.gd.md`
  - ゲーム進行、入力判定、タイマー、記録保存、背景機能を理解するための資料
- `docs/records.gd.md`
  - 記録一覧・詳細・評価、リプレイ、グラフ描画を理解するための資料

## よくある参照パターン
### 仕様確認したい
- 入力判定: `docs/main.gd.md` の「入力判定（状態機械）」
- お題フォーマット: `docs/main.gd.md` の「お題処理」
- 記録項目: `docs/main.gd.md` の「記録保存フォーマット」

### 不具合調査したい
- タイマーや開始/停止: `docs/main.gd.md` の「ラウンド制御」
- 記録が見えない/削除できない: `docs/records.gd.md` の「記録ファイルの読み書き」「一覧モード」
- リプレイが変: `docs/records.gd.md` の「リプレイ機能」「入力間隔グラフ」

### 変更するとき
- 記録スキーマを変える場合は、`main.gd` と `records.gd` をセットで更新する
- UI ノード名を変える場合は、対応する `find_child` / `ui_node` 参照を確認する

## 更新ルール（推奨）
- 実装を変更したら、同じコミットで該当ドキュメントも更新する
- 関数名・キー名・ファイルパスはコードと同じ表記で書く
- スキーマ変更時は影響範囲を明記する
